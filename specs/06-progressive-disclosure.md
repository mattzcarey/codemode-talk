# Slide 6: Progressive Disclosure of Tools

## Type
Concept / overview slide

## Content
- **Title**: "Progressive Disclosure"
- **Subtitle**: Don't show everything at once

- Three approaches presented as cards/columns:
  1. **CLI / Bash** — Self-discoverable, documented by design. `--help` is built in.
  2. **Tool Search** — BM25/semantic search over tool descriptions. Load tools on demand.
  3. **Code Mode** — Write code against a typed SDK. The SDK *is* the documentation.

- Each approach is a different philosophy:
  - CLI: "Let the agent explore"
  - Tool Search: "Let the agent find"
  - Code Mode: "Let the agent code"

## Layout
- Title top
- Three columns/cards below, each with an icon, name, and one-line description
- These will be expanded in slides 7-9

## Key point
- Tools should be discovered on demand, not dumped into context
- There are multiple valid approaches — we'll explore each
